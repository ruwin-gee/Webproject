Feedback Page Validation Report

Validated using the W3C Markup Validation Service at validator.w3.org

Reflection — feedback.html

During the first validation pass, aria-label was incorrectly placed on <div> elements for the review star ratings. These were changed to <p> tags. Programme card headings also used <h4> directly after <h2>, skipping a heading level — corrected to <h3>. After these fixes the page validated with no errors.

Team Page Validation Report

Validated using the W3C Markup Validation Service at validator.w3.org

Reflection — team.html

Validation flagged a warning that <section class="team-section"> lacked a heading. An <h2> was added inside the section to resolve this. A stray dash character after the mobile sidebar checkbox was also removed. After these corrections the page validated with no errors.

Content Page Validation Report

Validated using the W3C Markup Validation Service at validator.w3.org

Reflection — content_ST3.html

The content page validated cleanly after ensuring all images included alt attributes and all section elements contained at least one heading. The CSS-only contents toggle using a hidden checkbox validated without issues. No errors were present in the final build.
